## Prerequisites

- [Bun](https://bun.sh) ≥ 1.1 — Threaddy uses Bun as its runtime, package manager, bundler, and test runner
- Git

No Node.js, Docker, or other tools are required.

---

## Development setup

```bash
# Clone
git clone https://github.com/benedyktdryl/threaddy.git
cd threaddy

# Install dependencies
bun install

# Build the CSS (required before running the server)
bun run build:css

# Start the dev server (uses your real local data)
bun run dev
# → Server running at http://localhost:4821
```

> **Tip**: The dev server reads provider data from your machine the same way the production binary does. Run `threaddy init` first if you don't have a config file yet.

### CSS watch mode

While working on UI changes, run Tailwind in watch mode in a separate terminal:

```bash
bunx tailwindcss -i ./src/app/styles/globals.css -o ./src/app/assets/app.css --watch
```

## Project structure

```
src/
├── main.ts                   # Entry point
├── cli/main.ts               # CLI command dispatch
├── core/
│   ├── config/               # Config loading, schema, defaults
│   ├── logging/              # Structured logger
│   ├── types/domain.ts       # Shared domain types
│   └── utils/                # fs, id, text helpers
├── db/
│   ├── client.ts             # SQLite connection + migrations
│   ├── migrations/           # SQL migration files (embedded at build time)
│   └── repos/query-service.ts # All read queries
├── indexer/
│   ├── pipeline/indexer.ts   # Main indexing pipeline
│   └── watcher.ts            # File watcher + SSE sync
├── providers/
│   ├── claude-code/          # Claude Code transcript parser
│   ├── codex/                # Codex session parser
│   ├── cursor/               # Cursor composer parser
│   └── shared.ts             # Common parsing utilities
├── semantic-search/
│   ├── chunking/             # Text chunking strategies
│   ├── embeddings/           # HuggingFace transformers wrapper
│   ├── retrieval/            # Hybrid, semantic, keyword search
│   ├── storage/              # Embedding BLOB store
│   └── types/                # Search result types
├── app/
│   ├── assets/app.css        # Built CSS (gitignored, generated by tailwind)
│   ├── components/ui/        # shadcn-style UI primitives
│   ├── layout/app-shell.tsx  # Main HTML shell with sidebar
│   ├── routes/pages.tsx      # All page components (SSR)
│   ├── server/
│   │   ├── render.tsx        # React renderToString wrapper
│   │   └── router.tsx        # HTTP request router
│   └── styles/globals.css    # Tailwind source
└── test/                     # Test files (*.test.ts)
```

### Key architectural decisions

- **SSR only** — all HTML is rendered server-side with React. There is no client-side JavaScript bundle. Pages reload on navigation.
- **SQLite as the only store** — embeddings, metadata, full-text index, and state all live in one `.sqlite` file.
- **Embedded assets at compile time** — SQL migrations and the CSS file are bundled into the binary using Bun's `with { type: "text" }` and `with { type: "file" }` import attributes. No loose files needed at runtime.
- **Provider parsers are self-contained** — each provider module in `src/providers/` exports a scanner and a parser. Adding a new provider means adding a new directory there.

## Running tests

```bash
bun test
```

Tests live in `src/test/`. Each file is a `*.test.ts` using Bun's built-in test runner.

```bash
bun test --watch             # watch mode
bun test src/test/foo.test.ts  # single file
```

## Making changes

### Adding a provider

1. Create `src/providers/<name>/index.ts` implementing the `Provider` interface from `src/providers/base.ts`
2. Register it in `src/providers/registry.ts`
3. Add a config entry in `src/core/config/config-schema.ts` and `defaults.ts`
4. Write tests in `src/test/`

### Adding a DB migration

1. Create `src/db/migrations/00NN_description.sql` (increment the number)
2. Add an import and entry to `src/db/migrations/index.ts`

### Changing the UI

All page components are in `src/app/routes/pages.tsx`. The shared shell is in `src/app/layout/app-shell.tsx`. Styles use Tailwind with CSS variables defined in `src/app/styles/globals.css`.

After changing styles, rebuild CSS:

```bash
bun run build:css
```

## Submitting a pull request

1. **Fork** the repository and create a branch from `main`
   ```bash
   git checkout -b feat/my-feature
   ```

2. **Make your changes** — keep commits focused; one logical change per commit

3. **Run tests** and make sure they pass:
   ```bash
   bun test
   ```

4. **Test the build** to confirm the compiled binary still works:
   ```bash
   bun run build
   ./dist/threaddy stats
   ```

5. **Open a pull request** against `main`. Include:
   - What the change does and why
   - Any testing you did
   - Screenshots for UI changes

### Commit style

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add Cursor transcript body indexing
fix: handle missing project name in Claude Code parser
docs: add launchd setup example to README
chore: update @huggingface/transformers
```

## Reporting issues

Open an issue on [GitHub](https://github.com/benedyktdryl/threaddy/issues) with:

- Threaddy version (`threaddy --version` or from the release tag)
- OS and architecture (`uname -a`)
- What you expected vs. what happened
- Output of `threaddy doctor --json` (redact any paths you'd rather not share)
